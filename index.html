Welcome to the QrispUML wiki! Introduction This documentation is dedicated to the class diagrams provided at (insert) for Eclipse Qrisp. The documentation refers to the discovery of classes of Eclipse Qrisp and their respective UML class diagram, special features that were defined during the design process and additions if necessary. Eclipse Qrisp, hereinafter referenced by "Qrisp", is a high-level quantum programming language that provides many functions and quantum algorithms to the user. It provides a low-threshold interface for getting started with quantum programming and enables code execution on quantum backends. Qrisp implements 110 classes to this date (15.10.2025). Out of these 110 classes the source code was analyzed to determine datatypes and relationships. UML class diagrams are typically looking at classes from a statically typed viewpoint. Qrisp is implemented with the programming language Python as an embedded Domain Specific Language (eDSL) and is thus a dynamically typed programming language. This makes it challenging to find appropriate datatypes for function parameters, return values or properties of the class. Only with careful consideration and testing it was achievable to determine static datatypes. The classes are at times in relationship with each other but not specified further, this decision of design was taken to reduce redundancy across the diagrams. They summarize classes that were deemed as fitting, by either having a close relationship to each other or sharing similar concepts.
