Introduction

This documentation is dedicated to the class diagrams provided at QrispUML/documentation at main · Jfilz001/QrispUML for Eclipse Qrisp. The documentation refers to the discovery of classes of Eclipse Qrisp and their respective UML class diagram, special features that were defined during the design process and additions if necessary.

Eclipse Qrisp, hereinafter referenced by "Qrisp", is a high-level quantum programming language that provides many functions and quantum algorithms to the user. It provides a low-threshold interface for getting started with quantum programming and enables code execution on quantum backends.

Qrisp implements 159 classes to this date (12.05.2026). Out of these 159 classes the source code was analyzed to determine datatypes and relationships. UML class diagrams are typically looking at classes from a statically typed viewpoint. Qrisp is implemented with the programming language Python as an embedded Domain Specific Language (eDSL) and is thus a dynamically typed programming language. This makes it challenging to find appropriate datatypes for function parameters, return values or properties of classes. Only with careful consideration and testing it was achievable to determine static datatypes.

The classes are at times in relationship with each other but not specified further, this decision of design was taken to reduce redundancy across the diagrams.

The efforts accumulated to 11 class diagrams. They summarize classes that were deemed as fitting, by either having a close relationship to each other or sharing similar concepts. In the following listing are the categories in which classes have been placed.

  • Backends
  • Hamiltonian
  • Permeability
  • Operations
  • QuantumManagement
  • Tensors
  • QEnv & QPrimitive
  • QuantumVariables
  • Metrics
  • Algorithms and Problems
  • Other

The first chapter of this document is dedicated to provide a quick overview of the UML class diagrams listed above. The general purpose of the class diagrams is explained, and the described classes are defined. The second chapter goes into detail about the individual classes outlining design decisions, special annotations, and specific implementation details.

Hamiltonian

Qrisp provides a easy to use interface for simulating complex quantum behaviour of the total energy of systems, with the fundamental concept of Hamiltonians. They can be used to analyze complex interactions, for example simulating molecules in chemistry [1], but also have engineering [2], nanotechnology [3] and many more applications. The Hamiltonian UML class diagram shows all classes regarded to Hamiltonian simulation with its operations.

QEnvironment and QPrimitive

Qrisp supplies multiple environments that can be accessed with the python statement with. Qrisp treats these as if working with unmanaged resources, such as file streams. The environments are evaluated and result in an object that supports the context management protocol. For this the dunder methods ‘ __enter__’ and ‘__exit__’ must be individually specified for the classes. Additionally the ‘QuantumPrimitive’ class and descendants are specified as the ‘QuantumEnvironments’ inherit from it.

Algorithms and Problems

Algorithms and Problems In quantum computing, algorithms are structured as high-level problem classes that bundle a Hamiltonian or cost structure together with a quantum circuit approach or operator, an optional classical optimizer, and helper methods for running, training, and benchmarking. The classes listed below are implemented in Qrisp to provide complete quantum algorithm workflows across three families — VQE, QAOA/QIRO, and DCQO — plus the quantum backtracking framework. Note that these classes represent full algorithm problem instances rather than individual circuit operations. This description was derived from a Claude chat [38].
