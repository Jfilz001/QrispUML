Permeability

Information about classes and methods

The ‘Permeability’ diagram is a collection of classes that provide the ability to represent a permeability DAGs (Directed Acyclic Graph), describing a clear hierarchical execution of a task [17]. This facilitates numerous powerful transformations such as parallelization, memory management, synthesis of uncomputation and many more use-cases [18]. Permeability is a property of quantum gates, that was initially introduced by Frauenhofer Institute in [4] and gives a formal meaning of what “constant” could mean for a quantum function [18]. For a more detailed understanding of Permeability, here is is a Definition out of [4]:

Definition 2.1 (Permeability):

A (composite) quantum gate U is called Z-permeable in qubit i, if it commutes with the Z operator on this qubit.

U is permeable on qubit (1)

Similarly, U is called X-permeable in qubit i if it commutes with the X operator on this qubit

UnqompNode

Class functionality:

Described in the source code [14] as

“This is the base class of the Nodes that appear in the Permeability DAG”

It defines the core features of the Nodes for the Permeability DAG.

Class constructor and attributes:

The constructor receives and instantiates the ‘name’ String and ‘instr’ Instruction. Furthermore it defines an uninstantiated ‘UnqompNode’ as ‘uncomputed_node’. It also sets a unique id as a ‘hash’.

Class methods:

The ‘UnqompNode’ defines two methods: ‘_hash_’ a dunder method, which returns the hash value of the object and ‘_eq_’ a dunder method, which alters the functionality of the “==” operation by comparing the hash values of two ‘UnqompNodes’.

Details and design decision:

The ‘UnqompNode’ is the parent class of ‘InstructionNode’, ‘TerminatorNode’, ‘DeallocNode’ and ‘AllocNode’. It is the central class of the Permeability diagram.

The class constructor of ‘AllocNode’ initializes the ‘UnqompNode’ parent constructor and the bool attribute ‘artificial’.

Class methods:

‘AllocNode’ defines one method: ‘_repr_’ a dunder method, which changes the official string representation of an object.

Details and design decision:

‘AllocNode’ is the sub class of ‘UnqompNode’.

PermeabilityGraph

Class functionality:

Described in the source code [14] as:

“This class represents the Permeability Graph

As a subclass of networkx's DiGraph, it can be processed by

many of networkx's algorithms.“

Class constructor and attributes:

The class constructor of ‘PermeabilityGraph’ instantiates its parents class ‘nx.DiGraph’ constructor. Furthermore the only attribute ‘recent_node_dic’ is instantiated with the function ‘dag_from_qc’, which is from the same ‘permeability_dag.py’ file but not from the class. It is called with the received parameters ‘qc’ as a ‘Quantum Circuit’ and ‘remove_artificials’ as a bool, which is by default false.

Class methods:

The class methods provide functionality to visualize, convert, determining qubits, copying and many more operations with a permeability DAG.

Details and design decision:

Its parents class is ‘nd.DiGraph’. The ‘recent_node_dic’ maps classical bit or qubits to UnqompNode.
