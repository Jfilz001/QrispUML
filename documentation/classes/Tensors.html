Tensor

Information about classes and methods

The ‘Tensor’ diagram contains different array and tensor classes describing different mathematical structures, close to numpy arrays. Three major class emerge from this diagram: ‘DummyBiArray’, ‘DenseBiArray’ and ‘SparseBiArray’. The class ‘TensorFactor’ instantiates either one of these classes.

BiArray

Class functionality:

The ‘BiArray’ class is a central class in the ‘Tensor’ diagram. It provides the blueprint for the different types of arrays. The class does only provide very little functionality and is almost negligible

Class attributes:

The class has two attributes: an tuple of multiple integers ‘shape’ and an integer size.

Class methods:

The class has one method: ‘v_dot’, which returns either a ‘DummyBiArray’, ‘SparseBiArray’, ‘DenseBiArray’. It is a tensor dot of two ‘BiArray’ objects.

Details and design decisions:

The class does not have a class constructor. Due to this the class was labelled as abstract, even though the class does not inherit from the ‘ABC’ class. This design decision was made because the subclasses inherit the ‘BiArray’ functionality and it does not provide any functionality on its own, as its methods are not static.

DummyBiArray

Class functionality:

Out of the Qrisp sourcecode [14]:

“This class is used to estimate the complexity of a contraction order. The contract method does not actually execute a contraction but only logs the estimated amount of floating point operations.”

Class Constructor and Attributes:

The class constructor has 3 parameters. An optional ndarray ‘data’, an optional integer ‘shape’ and an integer ‘sparsity’, which is by default 1. The class attributes are:

  • ‘current_memory: ndarray’ – this 1 ndarray tracks active memory usage and only has 1 entry. It is static.
  • ‘max_memory: ndarray’ – this records the highest value of current_memory. It is static
  • ‘flops: ndarray‘ – this tracks the computational cost. It is static.
  • ‘sparsity: int‘ – this is the only attribute to define the properties of a specific ‘DummyBiArray’ object. It simulates the data density of the object (between 0 and 1)

Class methods:

The class contains 5 methods:

  • ‘reshape’ – modifies the dimensions (shape) of the current array instance to match the provided target shape.
  • ‘swapaxes’ – exchanges the positions of two specified axes within the array's shape tuple, effectively transposing those dimensions.
  • ‘contract’ – simulates a tensor contraction with another array by calculating the resulting shape and sparsity, while simultaneously updating the global resource counters (FLOPs and memory usage).
  • ‘copy’ – creates and returns a new ‘DummyBiArray’ instance that duplicates the current array's shape.
  • ‘reset_stat_counter’ – resets the shared global performance statistics (current_memory, max_memory, and flops) back to zero for all instances. The method is static

Details and design decisions:

The class inherits from ‘BiArray’. It has static attributes and methods, that are designed underlined.

DenseBiArray

Class functionality:

From the Qrisp documentation [14]:

“This class serves mainly as an interface to numpy arrays using the same methods as the SparseBiArray class this way the algorithm using these classes doesn't need to care about wether it's treating a sparse array or a dense array.

Furthermore, some multithreading techniques from the SparseBiArray class are deployed.”

Class constructor and attributes:

The class receives 3 parameters, of which 2 are optional. ‘array’ is a ndarray, ‘sparsity’ is an optional integer and ‘shape’ is an optional tuple of integers. The class attributes are:

‘data: ndarray’ – saves the array that has been called in the constructor.

‘dtype: numpy.dtype’ – the dtype of the data ndarray.

‘sparsity: float’ – out of the Qrisp code [14]: “If not given, estimate sparsity. In many situations the sparsity of a contraction result can be estimated (and then given to the constructor), so it is helpfull to track it.”

‘index_bit_permutation: List<Integer>’ – a list of integer indices based on the logarithm (base 2) of the objects size.

‘thread: Thread’ – this is a dynamically set attribute that means that the objects data is under construction.

Class methods:

The methods of ‘DenseBiArray’ are as follows:

  • ‘copy’ – this method copies the ‘DenseBiArray’ object
  • ‘catch_up’ – this method only works, when the object has a thread attribute. It joins the thread and depending on the time it took it will delete the thread attribute.
  • ‘reshape’ – this method reshapes when the data is required for a contraction
  • ‘swapaxes’ – this method swaps the two axes
  • ‘move_index_to_front’ – “This method allows moving an index to the first position without changing the order of the remaining indices” [14]
  • ‘apply_swaps’ – “This method applies the index permutation generated too” [14]
  • ‘ravel’ – this method applies the reshape and the apply_swaps method on the object and returns it
  • ‘contract’ – “identify contraction axes, move to the front, transpose one matrix and then apply matrix multiplication” [14]
  • ‘squared_norm’ – “Calculate the squared norm, ie. the sesquilinear scalar product of self with itself” [14]
  • ‘vdot’ – returns the dot product of two ‘DenseBiArray’ in form a ndarray
  • ‘exclude_linear_independence’ – this method returns a “cheap” guess whether two inputs are linearly independent. [14]
  • ‘to_array’ – this method simply return the ‘DenseBiArray’ as an ndarray
  • ‘to_sparse’ – this method converts the object into a ‘SparseBiArray’ and returns it
  • ‘__repr__’ – this dunder method changes the representation of the object

Details and design decisions:

This class has many similarities with the ‘SparseBiArray’ class. It can be instantiated by the ‘TensorFactor’ class and inherits from ‘BiArray’. The object can have a thread but does not have a thread necessarily, thus it is marked with “[0..1]” to indicate the possibility of both.

SparseBiArray

Class functionality:

From the Qrisp documentation [14]:

“This SparseBiArray class provides a numpy-esque rank n tensor interface, which uses sparse matrix multiplication for tensor contraction. The fact that only powers of two are allowed for the size, allow efficient index manipulations, that permit faster construction and processing of said sparse matrices compared to for instance the rank n tensor interface of the "sparse" package.”

Class constructor and attributes:

The class receives 3 parameters, of which 2 are optional. ‘init_object’ is either a ndarray an tuple of the form (ndarray, ndarray) or a scipy.sparse.coo_matrx. ‘shape’ is an optional tuple of integers. ‘contraction_counter’ is an optional integer, which is by default 0. The class attributes are:

  • ‘nz_indices: ndarray’ – this ndarray defines the non-zero indices
  • ‘contraction_counter’ – “This attribute decides after how many contractions, there should be a check for duplicates and zero entries in the sparse matrix data” [14]
  • ‘data: ndarray’ – saves the array that has been called in the constructor.
  • ‘_dict_’ – this dictionary (map) saves the values of the ‘init_object’s dictionary. This attribute only exists, if the ‘init_object’ is a ‘SparseBiArray’
  • ‘sparsity: float’ – out of the Qrisp code [14]: “If not given, estimate sparsity. In many situations the sparsity of a contraction result can be estimated (and then given to the constructor), so it is helpfull to track it.”
  • ‘index_bit_permutation: List<Integer>’ – a list of integer indices based on the logarithm (base 2) of the objects size.
  • ‘dtype: numpy.dtype’ – the dtype of the data ndarray.
  • ‘thread: Thread’ – this is a dynamically set attribute that means that the objects data is under construction.

Class methods:

The methods of ‘SparseBiArray’ are as follows:

  • ‘copy’ – this method copies the ‘SparseBiArray’ object
  • ‘catch_up’ – this method only works, when the object has a thread attribute. It joins the thread and depending on the time it took it will delete the thread attribute.
  • ‘reshape’ – this method reshapes when the data is required for a contraction
  • ‘swapaxes’ – this method swaps the two axes
  • ‘apply_swaps’ – “This method applies the index permutation generated [...]” [14]
  • ‘sum_duplicates’ – “This method sums duplicates ie. if an index appears twice in nz_indices the corresponding data entries are summed.” [14]
  • ‘elim_zeros’ – “If after a contraction a new zero appeared, this entry does not need to tracked anymore. This method removes the zeros” [14]
  • ‘build_sr_matrix’ - “Builds up the sparse matrix in coo format. The coo format saves an array of row/column indices and the corresponding entries. Shape is the shape the sparse matrix should have” [14]
  • ‘move_index_to_front’ – “This method allows moving an index to the first position without changing the order of the remaining indices” [14]
  • ‘contract’ – “identify contraction axes, move to the front, transpose one matrix and then apply matrix multiplication” [14]
  • ‘to_array’ – this method simply returns the ‘SparseBiArray’ as an ndarray
  • ‘ravel’ – this method applies the reshape and the apply_swaps method on the object and returns it
  • ‘__repr__’ – this dunder method changes the representation of the object
  • ‘split’ – “This function allows to split the array into it's lower and upper half […]” [14]
  • ‘multi_measure’ - This method performs a multi-index measurement by computing all non-zero measurement outcomes, their probabilities, and optionally the corresponding post-measurement collapsed sparse states.
  • ‘exclude_linear_independence’ – this method returns a “cheap” guess whether two inputs are linearly independent. [14]
  • ‘squared_norm’ – “Calculate the squared norm, ie. the sesquilinear scalar product of self with itself” [14]
  • ‘vdot’ – returns the dot product of two ‘SparseBiArray’ in form a ndarray
  • ‘to_dense’ – this method converts the object into a ‘DenseBiArray’ and returns it.
  • ‘to_sparse’ – this method returns the object.

Details and design decisions:

This class has many similarities with the ‘DenseBiArray class. It can be instantiated by the ‘TensorFactor’ class and inherits from ‘BiArray’. The object can have a thread and a ‘__dict__’ but not necessarily, thus they are marked with “[0..1]” to indicate the possibility of both.

TensorFactor

Class functionality:

The ‘TensorFactor’ class represents a factorized quantum state over a subset of qubits. The class is designed to minimize memory movement and copying, making it critical for efficient large-scale quantum simulation.

Class constructor and attributes:

The class constructor receives 2 parameters, of which 1 is optional. ‘qubits’ is described in the Qrisp source code as follows:

“This list contains a list of integers, which describe the current permutation of qubit indices inside this TensorFactor, i.e. swapping two elements of this list is the same as applying a swap gate. Using this structure is advantageous because applying matrices on specific qubit indices requires alot of swapping. Swapping the indices of an array is ressource costly because the array needs to be copied. As it turns out memory speed is a bottleneck.” [14]

The second parameter is ‘init_tensor_array’, which is an optional ‘BiArray’ instance. IT could be either a ‘DummyBiArray’, a ‘DenseBiArray’ or a ‘SparseBiArray’. It provides the ‘tensor_array’ that is operated upon.

The class attributes are:

  • ‘qubits: List<Qubit>’ – this is has been described above
  • ‘n: int’ – this is the amount of qubits
  • ‘tensor_array: BiArray’ – this is the ‘BiArray’ that is being operated on.
  • ‘index_init: ndarray’ – represents an index into a sparse tensor, this is a static attribute
  • ‘data_init: ndarray’ - represents the corresponding tensor index, this is a static attribute

Class methods:

The methods of ‘TensorFactor’ are as follows:

  • ‘copy’ - this method copies the SparseBiArray object and returns a deep copy of the TensorFactor
  • ‘swap’ - this method swaps two qubit indices and updates the underlying tensor accordingly
  • ‘apply_matrix’ - this method applies a matrix to the specified qubits by reshaping and contracting the tensor-
  • ‘entangle’ - this method entangles the current TensorFactor with another disjoint TensorFactor-
  • ‘measure’ - this method measures a single qubit and returns outcome probabilities and resulting tensor factors
  • ‘unravel’ - this method reorders the qubits into ascending order using axis swaps
  • ‘multi_measure’ - this method measures multiple qubits and returns probabilities, tensor factors, and outcome indices
  • ‘disentangle’ - this method attempts to separate a qubit from the tensor factor and returns independent factors when possible

Details and design decisions:

The ‘TensorFactor’ class has two static attributes, which are underscored. The class instantiates a ‘tensor_array’, which can be either a ‘DummyBiArray’, ‘DenseBiArray’ or a ‘SparseBiArray’. Therefore there are three aggregations.
