Metrics

Information about classes and methods

The ‘ Metrics’ diagram is named after the central class of the diagram: ‘BaseMetric’. It unites the different Metrics with its descendants, outlining the class hierarchy. Different implementations of ‘ BaseMetric’ can be found in this diagram.

BaseMetric

Class functionality:

As specified in the code [34]:

“Runtime-enforced base class for profiling metrics. Classes inheriting from ‘BaseMetric’ must implement handler methods for all quantum primitives, and define an ‘initial_metric’ property that specifies the initial value of the metric before any primitives are executed.”

Class constructor and attributes:

The ‘BaseMetric’ class has a constructor, though the class inherits from ‘ABC’, thus making it abstract. The class constructor is used for sub-classes. The class has one attribute: ‘meas_behavior’, a callable function, which is a parameter given for the class constructor. It is the measurement behavior function, as the name suggests.

Class methods:

The class lays the blueprint for subclass by supplying many methods. They are annotated with the ‘@abstractmethod’ decorator. The class also implements internal helper methods and static methods.

The abstract methods are:

- ‘from_cache_key’ – “Reconstructs a metric instance from a hashable cache key.” [34]. This method is static and thus designed with a underlining.

- ‘initial_metric’ – “Return the initial value of the metric before any primitives are executed." [34]

- ‘cache_key’ – “Return a hashable representation of this metric's configuration.” [34]

- ‘handle_create_qubit’– “ Handle the `jasp.create_qubits` primitive. The `create_qubits_p` primitive has the following semantics:

o - Invars: (size, QuantumState)

o Outvars: (QubitArray, QuantumState)”[34]

- ‘handle_get_qubit’ – “ Handle the `jasp.get_qubit` primitive. The `get_qubit_p` primitive has the following semantics:

o Invars: (QubitArray, index)

o Outvars: (Qubit)” [34]

- ‘handle_get_size’ – “ Handle the `jasp.get_size` primitive. The `get_size_p` primitive has the following semantics:

o Invars: (QubitArray)

o Outvars: (size)” [34]

- ‘handle_fuse’ – “ Handle the `jasp.fuse` primitive. The `fuse_p` primitive has the following semantics:

o Invars: (Qubit | QubitArray, Qubit | QubitArray)

o Outvars: (QubitArray)” [34]

- ‘handle_slice’ – “ Handle the `jasp.slice` primitive. The `slice_p` primitive has the following semantics:

o Invars: (QubitArray, start, stop)

o Outvars: (QubitArray)“ [34]

- ‘handle_quantum_gate’ – “Handle the `jasp.quantum_gate` primitive. The `quantum_gate_p` primitive has the following semantics:

o Invars: (Qubit 0, Qubit 1, ... , Param 0, Param 1 ... , QuantumState)

o Outvars: (QuantumState)“ [34]

- ‘handle_measure’ – “ Handle the `jasp.measure` primitive. The `measure_p` primitive has the following semantics:

o Invars: (Qubit | QubitArray, QuantumState)

o Outvars: (meas_result, QuantumState)“ [34]

- ‘handle_reset’ – “ Handle the `jasp.reset` primitive. The `reset_p` primitive has the following semantics:

o Invars: (QubitArray, QuantumState)

o Outvars: (QuantumState)“[34]

- ‘handle_delete_qubits’ – “Handle the `jasp.delete_qubits` primitive. The `delete_qubits_p` primitive has the following semantics:

o Invars: (QubitArray, QuantumState

o Outvars: (QuantumState)”[34]

- ‘handle_parity’ – “Handle the `jasp.parity` primitive.”[34]

The internal helper methods are:

- ‘_validate_measurement_result’ – "Validate that measurement result is a boolean." [34]

- ‘_measurement__body_fun’ – “Helper function for measuring qubit arrays.” [34]

The normal methods are:

- ‘handle_create_quantum_kernel’ – “Handle the `jasp.create_quantum_kernel` primitive.”[34]

- ‘get_handlers’ – “Return a mapping from primitive names to handler methods.” [34]

Details and design decisions:

The class has internal helper methods, that are designed as protected(#). The class is inheriting from ‘ABC‘, thus its title is written in cursive, as well as all of the subsequent abstract methods. Static methods (class methods) are designed with a underlining. The handlers can return different datatypes, this is done with the ‘Sequence’ datatype. It is not possible to map certain datatypes. Its child classes are:

- ‘CountOpsMetric’

- ‘DepthMetric’

- ‘NumQubtisMetric’
