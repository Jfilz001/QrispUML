Hamiltonian

Information about classes and methods:

The Hamiltonian class itself is an abstract class. It can be implemented by one of its 3 subclasses. All of them have a ‘terms_dict’ dictionary as the only class Attribute but provide many methods, particularly many methods that alter the syntax of the eDSL.

Hamiltonian

Class functionality:

“Central structure to facilitate treatment of quantum Hamiltonians.” [14] This function provides the blueprint for implementation of the Hamiltonian datatypes.

Class constructor and methods:

The ‘Hamiltonian’ class constructor and methods are not implemented, as they simply pass. As stated previously, the ‘Hamiltonian’only provides an abstract interface of a Hamiltonian datatype. Many methods alter the syntax, labelled with the < annotation. Alteration of representations are labelled with the < annotation. The methods are abstract, which is represented in italics in the diagram.

Details and design decision:

‘Hamiltonian’ is the parent class of ‘FermionicOperator’, ‘BoundQubitOperator’, ‘QubitOperator’. It inherits from the ‘ABC’ module for abstract classes.

FermionicOperator

Class functionality:

“This class provides an efficient implementation of ladder term operators, i.e., operators of the form

where each term is a product of fermionic raising and lowering operators acting on the fermionic mode.” [15]

Class constructor and attributes:

The class constructor of ‘FermionicOperator’ instantiates the ‘terms_dict’ dictionary, as empty by default. The Datatypes of the keys and values are undetermined and thus are referred to as Object.

Class methods:

The class implements many methods that change the object of the class and return a new ‘FernionicOperator’. ‘Trotterization’ and ‘to_qubit_operator’ are also examples of the many uses this class brings.

Details and design decision:

Some class methods can be called, without instantiating the class, such as the ‘from_openfermion’ method. They are written with underscores in the documentation. The class has a method called ‘from_pyscf’, which takes a pyscf object. The document specifies it as ‘pyscf.gto.mole.Mole‘ [15]. The class makes use of complex integers, which are simply referred to as ‘complex’. Its parent class is the abstract class ‘Hamiltonian’.

BoundQubitOperator

Class functionality:

“This class provides an efficient implementation of Pauli Hamiltonians acting on ‘QuantumVariable’s, i.e., Hamiltonians“ [15]

Class methods:

The class implements many methods that change the object of the class and return a new ‘BoundQubitOperator’. ‘Trotterization’ and ‘ground_state_energy’ are also examples of the many uses this class brings.

Details and design decision:

The method ‘trotterization, implements multiple functions and uses all of these functions to return a function. The class makes use of complex integers, which are simply referred to as ‘complex’. The ‘measurement_data’ parameter of the method ‘get_measurement’ is most likely a dictionary that maps a bitstrings to integers, while it is hard to tell, this is why its datatype is Object. The parent class of ‘BoundQubitOperator’ is the abstract class ‘Hamiltonian’.

QubitOperator

Class functionality:

“This class provides an efficient implementation of QubitOperators, i.e. Operators, that act on a qubit space . “ [16]

Class methods:

The class implements many methods that change the object of the class and return a new ‘QubitOperator‘. With methods such as ‘to_array‘ or ‘to_sparse_matrix’ the class provides functionality to convert ‘QubitOperator’ objects into different representations.

Details and design decision:

Some class methods can be called, without instantiating the class, such as the ‘from_numpy_array’ method. They are written with underscores in the documentation. The class makes use of complex integers, which are simply referred to as ‘complex’. The parent class of ‘QubitOperator’ is the abstract class ‘Hamiltonian’.
